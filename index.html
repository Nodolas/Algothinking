K-Means
Clustering
Finding Hidden Groups

An unsupervised machine learning algorithm that discovers natural structure in unlabelled data — no human guidance required.

Algorithm Type Unsupervised ML
Time Complexity O(n · k · d · i)
Proposed by Lloyd, 1957
Core Paradigm Iterative Optimisation
01 The Problem

Data arrives without labels.
Structure hides inside it.

Imagine you're handed a spreadsheet of 50,000 customers. No categories, no tags — just numbers: purchase frequency, average spend, last login, items browsed. Your job: find the natural groups.

You could hire an analyst to read every row and decide — but that doesn't scale. You could pick rules manually ("anyone who spends over €200 is 'high value'") — but you're imposing your assumptions on data that may tell a different story.

❌ Naive Approach

Manual rule-setting

A human defines categories up front. Works for 100 rows. Completely breaks at 100,000. Misses patterns that weren't anticipated. Requires domain expertise for every dataset.

✓ K-Means

Let the data speak

The algorithm finds natural groupings mathematically. Scales to millions of points. Discovers unexpected patterns. Runs in minutes on any modern machine.

The core question K-Means answers: Given N unlabelled data points and a number K, what are the K most natural groupings — and which group does each point belong to?

Why can't we just measure all pairwise distances?

For N=10,000 points, brute-force pairwise comparison means 50 million distance calculations — just to find the most similar pair. K-Means solves this in O(N·K) per iteration, which is dramatically cheaper. The trade-off: we must choose K in advance and accept a local (not always global) optimum.

02 The Core Idea

Three words:
Assign. Relocate. Repeat.

The algorithm works like a city planner trying to place K emergency stations to minimise the maximum travel time for any resident. You don't know the perfect locations — so you start somewhere and improve.

The key insight: if you knew which points belonged to each cluster, you could immediately find the best centre (the mean). And if you knew the centres, you could immediately assign every point. K-Means alternates between these two steps until they agree.

📍

Initialise

Place K centroids at random positions in the data space. Bad starting points are fine — we'll fix them.

🔗

Assign

Each data point joins the cluster whose centroid is nearest (Euclidean distance). No point is left unassigned.

🎯

Update

Each centroid moves to the arithmetic mean of its assigned points. Repeat Assign → Update until nothing changes.

What the algorithm minimises: Within-Cluster Sum of Squares (WCSS) — the total squared distance from every point to its assigned centroid. Lower WCSS = tighter, more meaningful clusters.

The mathematical objective

J = Σₖ Σᵢ∈Cₖ ‖xᵢ − μₖ‖²

Where xᵢ is a data point, μₖ is the centroid of cluster k, and Cₖ is the set of all points assigned to cluster k. We want to minimise J — the algorithm does this by alternating between the two steps above, each of which is guaranteed to decrease J or leave it unchanged.

05 Complexity & Trade-offs

Fast in practice.
Tricky in theory.

Metric Value What the variables mean
Time Complexity O(n · k · d · i) n = points, k = clusters, d = dimensions, i = iterations
Space Complexity O(n · k) Store distance from each point to each centroid
Convergence Guaranteed finite WCSS decreases or stays equal every iteration; finite states
Optimality Local minimum only May miss the global optimum; run multiple times with different seeds
Typical iterations 10–100 Usually converges quickly in practice, despite worst-case being exponential

When it shines — and when it fails

⚡

Strength: Scales well

Linear in the number of points per iteration. Handles millions of data points efficiently with mini-batch variants.

🔍

Strength: Easy to interpret

Each cluster has a centroid you can describe in plain language. Results are immediately actionable without specialist knowledge.

🎲

Weakness: K must be chosen

You decide K up front. Use the Elbow Method (plot WCSS vs K) or Silhouette Score to find a good K empirically.

🔵

Weakness: Assumes round clusters

K-Means uses distance to assign points, so it assumes roughly spherical, similarly-sized clusters. Fails on crescents or rings.

💥

Weakness: Sensitive to outliers

A single extreme outlier pulls its centroid far from the true cluster mean. Consider K-Medoids (PAM) for robust alternatives.

🔧

Fix: K-Means++

Smart initialisation picks starting centroids far apart, dramatically reducing bad convergence and cutting required iterations.

Elbow Method — choosing K: Run K-Means for K=1,2,3,...,10. Plot WCSS against K. The "elbow" — where adding another cluster stops meaningfully reducing WCSS — is your best K.
06 Company Application

Two concrete
real-world uses.

01
Application · Customer Intelligence

Behavioural Customer Segmentation for Personalised Marketing

An e-commerce company holds transaction data for 200,000 customers: purchase frequency, average order value, product category mix, days since last purchase, and session duration. Currently all customers receive the same weekly newsletter — conversion rate: 1.8%.

Applying K-Means (K=5) to this data discovers distinct segments: frequent high-spenders, seasonal buyers, lapsed customers, price-sensitive deal-hunters, and new experimenters. Each segment receives targeted messaging: lapsed customers get win-back offers; deal-hunters get early sale access; high-spenders get loyalty perks.

Specific implementation: cluster on normalised (z-scored) features [recency, frequency, monetary, category_entropy]. Run K-Means++ with K=5, validate using Silhouette Score ≥ 0.45. Feed cluster labels into the email platform as audience segments. Re-cluster monthly as behaviour evolves.

02
Application · Data Quality

Detecting Anomalous Entries in a Parts / SKU Database

A manufacturing company's internal parts database contains 50,000 SKUs with inconsistently formatted names: "Bolt M8×30 SS", "M8 × 30mm Stainless Bolt", "stainless_bolt_8mm_30", "BOLT-SS-M8-30". This causes duplicate orders, incorrect inventory counts, and sourcing errors worth tens of thousands of euros per quarter.

K-Means on character-level TF-IDF embeddings of the SKU names clusters visually similar strings together. Cluster centroids represent the "canonical" format. Items far from their centroid are outlier descriptions of likely-duplicate items — flagged for a human review queue. This shrinks the manual review burden from 50,000 entries to ~1,200 flagged anomalies.

Specific implementation: vectorise SKU names with TF-IDF (character 3-grams). Apply K-Means with K estimated from product taxonomy (~800 part families). Points in the 95th percentile of distance-to-centroid are flagged. Cross-reference against existing Levenshtein matching for confirmation.

One-pager to keep.

Algorithm Cheat Sheet

K-Means Clustering

Lloyd (1957)
Complexity: O(n·k·d·i)
Type: Unsupervised ML
Core Idea

Partition N points into K clusters by iteratively assigning each point to its nearest centroid, then recomputing centroids as the mean of assigned points. Stop when assignments no longer change.


The Four Steps
  • Initialise K centroids (randomly or K-Means++)
  • Assign each point to nearest centroid
  • Move centroid to mean of its group
  • Repeat until convergence
Mini Example (K=2)

Points: A(1,2) B(2,1) C(8,8) D(9,9)
Init: C1=(1,2) C2=(9,9)
Assign: {A,B}→C1, {C,D}→C2
Update: C1=(1.5,1.5) C2=(8.5,8.5)
Re-assign: unchanged → converged ✓


Objective (minimise)

J = Σₖ Σᵢ∈Cₖ ‖xᵢ − μₖ‖²

When to use
  • Customer segmentation
  • Image colour quantisation
  • Document clustering
  • Anomaly detection
  • Feature engineering

When NOT to use
  • Non-spherical clusters
  • Unknown / variable K
  • Many outliers present
  • Very high dimensions
Better alternatives for edge cases
K-Means++ · K-Medoids · DBSCAN · Gaussian Mixture Models · Hierarchical Clustering
Choose K using
Elbow Method · Silhouette Score · Gap Statistic · BIC / AIC
